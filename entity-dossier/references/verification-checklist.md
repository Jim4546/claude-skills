# Verification Checklist

Run this before sending the Feishu/Lark link.

## Identity

- Company legal subject is confirmed by registry or equivalent.
- Person identity is tied to at least two anchors when names are ambiguous.
- Core company-person relationship is sourced and dated where possible.
- User-requested exclusions are absent from narrative, diagrams, and source appendix unless necessary for disambiguation.

## Facts

- No unsupported financing, investor, product, customer, title, school, or relationship claims.
- All dates use the precision shown by the source.
- Person external appointments include start and end/status when public data supports it.
- Ordinary gaps are handled as `公开资料未显示` or omitted; only core identity/relationship uncertainty triggers a user question.

## Equity

- Direct holdings match the source.
- Indirect holdings show path and formula.
- Decimal rounding is consistent, preferably 4 decimal places for percentages unless source precision differs.
- Person totals equal direct plus all confirmed indirect paths.

## Feishu/Lark Document

- Title follows `{公司简称}与{人物名}信息整理YYYYMMDD`.
- Main modules are 核心结论、公司信息、人物背景、附录：信息来源 unless the user asked otherwise.
- Tables are readable and dense.
- Diagrams do not introduce facts absent from the text.
- Source appendix is last and includes all sources used.

## Final Readback

- Reopen/read the created document or exported content when tools allow it.
- Check that the source count, headings, and diagram count match the intended deliverable.
- If publishing to a wiki node, verify the final URL is the wiki/doc URL the user can open.
