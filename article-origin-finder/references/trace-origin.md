# 溯源 + 核实分级

拿到正文后，目标是**判定真相**：这篇是不是某英文原文的转载/编译？原文是什么、在哪、内容确切如何？哪些是译者的再加工？

核心原则：**轻量优先，按需升级**。不要一上来就全网扇出烧 token。

## 第 1 步 · 自报出处优先（覆盖大多数情况）

`fetch_article.py` 的 JSON 已经给了 `source_links` 和 `source_context`。先看它们：

- 文末"原文链接 / 来源 / 编译自 / 译自 / source / via"后面的 URL，就是作者自报的出处。
- 顺着它取原文：先 WebFetch，被墙就按 [scrape-ladder.md](scrape-ladder.md) 升级。
- 原帖锁登录（X 长文等）→ 用原文开头的 distinctive 句子，搜**全文转载源 / 新闻复述**取得可读全文，再核。

> 本 skill 来历的案例：微信文末直接列了两个 X 链接。第一个 `x.com/satyanadella/...` 是 Satya Nadella 长文《A frontier without an ecosystem is not stable》——这就是英文原文；正文锁登录，最终在一个全文转载站拿到逐字全文。**第二个链接其实是一条 AI 量化交易营销贴（被 X 社区笔记标注不实），并非原文配套**——所以自报出处也要核，不能照单全收。

## 第 2 步 · 升级到多源扇出（仅在必要时）

**触发条件**（满足任一）：
- 正文没有任何自报出处；
- 自报出处取不到 / 核不实 / 与正文对不上；
- 用户明确要求"深挖 / 彻底核实"。

**做法**（可借助 Workflow 工具并行，参考本 skill 来历那次的形态）：
1. 抽取正文里的 **distinctive 短语 / 专有名词 / 人名 / 独特提法**（英文概念尤其有用）。
2. 多角度并行搜：① 原帖+镜像 ② distinctive phrase 直搜 ③ 新闻/博客/LinkedIn/Substack 复述 ④ 事件真实性核查。
3. 各源回报候选 URL + 命中片段 → 去重 → 取全文最完整的源做逐字依据。
4. **绝不编造**：只用真正抓到的英文。拿不到全文就如实标"partial / 仅片段"。

## 第 3 步 · 判定真相（含"无英文原文"分支）

- **有英文原文** → 进入核实分级（下），整理三层文档。
- **原生中文 / 查无此原文 / 自报出处造假** → 明确产出结论：**"未发现英文原文"**，说明依据（搜遍 distinctive 短语无果 / 出处链接指向无关内容 / 内容特征是中文原创）。**不要硬编一篇译文交差。**

## 核实分级（中英核对表的判定标准）

逐条比对中文稿与英文原文，每条标注一档：

| 档 | 含义 | 判据 |
|-|-|-|
| **真实·逐字** | 中文忠实对应原文该句 | 能在英文原文找到 verbatim 出处 |
| **意译** | 概念对、但措辞是译者再造 | 原文有对应意思，但中文用了不同比喻/说法（例：原文 "hill climbing machine" 被译成"进化机器/学习阶梯"；"stable equilibrium" 被译成"动态均衡"） |
| **虚构·未证实** | 原文/可靠来源里查无此项 | 多源都找不到佐证（例："90 分钟整改窗口"无任何来源） |
| **再加工** | 把无关事实强行编织成因果/叙事 | 两件独立的事被作者拼成因果（例：把 Nadella 备忘录与某模型出口禁令绑成同一叙事，实则毫无关联） |

> 即使是**完全忠实**的译文，核对表也要保留，并写明一句"逐句忠实、无虚构、无再加工"——让"已核实"这件事本身成为交付物的一部分。

## 事件真实性 ≠ 叙事真实性

注意区分两层：
- **底层事实**可能为真（如某模型确实发布、确实受某监管影响）；
- **作者的叙事框架**可能是再加工（夸张的时间线、强加的因果、误导性引用）。
核对表要把这两层分开标，别因为"底层事实为真"就整体判定文章可信。
