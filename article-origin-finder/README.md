# article-origin-finder

文章原文溯源 skill。给定一篇文章 URL，它会抓取正文、寻找原始出处，优先定位英文原文，并把中文稿和原文逐条核对，最后整理成飞书云文档；飞书不可用时会降级为本地 Markdown。

## 适合什么场景

- 中文文章疑似转载、编译或二次加工，需要找到英文原文。
- 微信公众号、媒体站点或博客文章需要做真实性核查。
- 需要把“中文稿、英文原文、差异核对表”整理成可交付文档。
- 需要判断文章是否没有英文原文，或所谓出处是否编造。

## 工作流程

1. 抓取正文：先普通抓取；遇到反爬、登录墙、微信环境异常等情况，再升级为本机浏览器抓取。
2. 寻找出处：解析文末链接、原作者、编译说明、引用标题和关键词。
3. 多源核实：用搜索、交叉引用和正文片段比对确认原文。
4. 中英核对：标注逐字翻译、意译、删改、未证实或虚构内容。
5. 产出文档：默认写入飞书；没有飞书权限时生成本地 Markdown。

## 一次性准备

浏览器抓取需要 Playwright：

```bash
pip install playwright
playwright install chromium
```

写入飞书需要 `lark-cli` 并完成登录：

```bash
lark-cli auth login --domain docs
```

如果原文是视频或播客，可能还需要字幕工具：

```bash
pip install yt-dlp
```

## 常用说法

- “找一下这篇文章的英文原文并整理进飞书：<URL>”
- “这篇是哪来的？帮我溯源核实：<URL>”
- “帮我查证这篇中文稿是不是编译稿：<URL>”
- “/article-origin-finder <URL>”

## 输出

飞书文档默认包含三层：

1. 原文信息表：中文稿、原文、作者、来源、发布时间、可信度。
2. 英文原文全文：可确认来源的原文内容或逐字稿。
3. 中英核对表：逐段标出一致、意译、删改、缺失和未证实内容。

如果文章本来就是中文原创，或没有可靠英文原文，skill 会明确标注“未找到英文原文”，不会硬编来源。

## 注意事项

- 仅用于合理引用、研究和事实核查。
- 抓取使用本机网络和浏览器环境，请遵守目标站点使用条款。
- 受版权保护的内容应以引用、摘要和来源标注为主，不做不必要的全文转载。
