# wechat2lark 用户手册

> 在 Claude Code 里贴一个公众号链接，自动转成飞书云文档，归档到团队知识库。

## 一分钟用起来

```
帮我把这篇公众号转到知识库：https://mp.weixin.qq.com/s/abcdef
```

Claude Code 会自动触发本 skill，跑完后给你一个飞书 wiki 链接。

仅当你已经完成「安装」+「配置」两步。下面分别说。

---

## 1. 安装（每人一次）

### 1.1 前置依赖

| 依赖 | 怎么装 / 检查 |
|---|---|
| **Claude Code** | 公司应该都装了，没装去 https://claude.com/code |
| **lark-cli** | `npm install -g @larksuite/cli`，然后 `lark-cli auth login` 登录飞书。如果你之前用过 lark-* 系列 skill，跳过这步 |
| **Python 3.10+** | `python --version` 检查；没有就装 [Python 官网](https://www.python.org/downloads/)，安装时勾上「Add to PATH」 |
| **Python 包** | `pip install requests beautifulsoup4 python-docx` |

### 1.2 装 skill

```bash
npx skills add Jim4546/wechat2lark
```

跑一次就好。装完会在 `~/.claude/skills/wechat2lark/` 下出现 skill 目录。

验证：在 Claude Code 里随便聊一句，看 skill 列表里有没有 `wechat2lark`。

---

## 2. 配置（每人一次）

### 2.1 拿到团队共享知识库的信息

向 **skill 维护者**（一般是搭建这个 skill 的人）要两个值：

- `wiki_space_id`：团队共享知识库的 ID，一长串数字
- `parent_node_token`：归档的父节点 token，字母数字混合

> 当前团队的值（搭建者直接贴给队友）：
> - `wiki_space_id`：`7644455953701718997`
> - `parent_node_token`：`YCTywRYmwizGRAkg1AzchdZJnXf`
> - 知识库：「公众号文章归档」→「归档」节点

### 2.2 创建 config.json

进到 skill 目录：

```bash
cd ~/.claude/skills/wechat2lark
```

把示例配置拷一份：

```bash
# Mac / Linux
cp config.example.json config.json

# Windows PowerShell
Copy-Item config.example.json config.json
```

用编辑器打开 `config.json`，填进去：

```json
{
  "wiki_space_id": "7644455953701718997",
  "parent_node_token": "YCTywRYmwizGRAkg1AzchdZJnXf",
  "lark_identity": "user"
}
```

保存。一次性动作，之后不用再动。

### 2.3 把自己加进知识库

找 skill 维护者把你加进「公众号文章归档」知识库（不然 skill 写入会权限拒绝）。

## 3. 使用

### 3.1 标准用法

在 Claude Code 里说：

```
帮我把这篇公众号转到知识库：
https://mp.weixin.qq.com/s/LhUtw7ITAgNEKub6kFd8lQ
```

或者更简略：

```
转一下 https://mp.weixin.qq.com/s/xxxxx
```

Claude 会自动识别公众号链接并触发 skill。跑完后会输出一个 wiki 链接，点进去就能看到归档好的飞书文档。

### 3.2 文件命名规则

`{文章标题}_{发布日期 YYYYMMDD}`

例如：`华为Fellow胡欣蔚：AI Agent崛起！CPU逆袭、操作系统重构_20260525`

如果文章页解析不到发布日期，会用今天的日期兜底。

### 3.3 临时换个地方归档

不想归档到默认位置（比如想存到自己个人的笔记空间）：

```
把 https://mp.weixin.qq.com/s/xxxxx 转到这个 wiki：
space=<另一个 space_id>，node=<另一个 node_token>
```

Claude 会用这次给的参数覆盖 config 里的默认值。

## 4. 转换效果说明

**会保留**：
- ✅ 标题、作者、发布日期、原文链接（首页元信息）
- ✅ 段落、标题（h1-h4）
- ✅ 图片（自动 embed 到 docx，不依赖外链）
- ✅ 列表（有序 / 无序）
- ✅ 引用块、代码块
- ✅ 表格（基础形式）

**不会保留 / 会简化**：
- ❌ 公众号的复杂排版样式（彩色框、特殊字体、动画）
- ❌ 视频卡片（变成占位文字）
- ❌ 小程序 / 链接卡片（变成占位文字）
- ❌ 内嵌的音频（忽略）

**完全做不到**：
- ❌ 需要「关注后可见」的私密文章——会报错「正文为空」

## 5. 常见问题

### Q1. 报错「config_missing」

→ 你跳过了第 2 步配置。回去拷 `config.example.json` 为 `config.json` 并填值。

### Q2. 报错「lark-cli failed: ... unauthorized / 403」

→ 两种可能：
1. `lark-cli` 没登录或登录过期：`lark-cli auth login` 重新登录
2. 你没被加进共享知识库：找维护者把你 `wiki +member-add` 加进去

### Q3. 报错「WeChat returned an anti-abuse page」

→ 短时间内转多篇，被微信临时限流了。等 5-10 分钟再试，或换个网络。**不要疯狂重试，会被封更久。**

### Q4. 报错「Could not find #js_content. The article may require login」

→ 文章是「关注后可见」或已被作者删除。打开链接如果浏览器里也看不到正文，本 skill 就没办法。

### Q5. 跑完了但 wiki 链接打开是 404

→ 链接用的是通用 `feishu.cn/wiki/<token>`，应该会自动跳到 tenant 域名。如果没跳，手动把 URL 改成 `https://你们公司的tenant.feishu.cn/wiki/<token>`。

### Q6. 排版不满意

→ 公众号 HTML 里有很多嵌套 `<section>` 样式，docx 没法完全保真。如果某类内容（比如代码、引用）转出来效果很差，提个 GitHub issue：https://github.com/Jim4546/wechat2lark/issues

### Q7. 想批量转一堆链接

→ 直接给 Claude 多个链接：

```
帮我把这几篇都转到知识库：
1. https://mp.weixin.qq.com/s/aaa
2. https://mp.weixin.qq.com/s/bbb
3. https://mp.weixin.qq.com/s/ccc
```

Claude 会逐个调用 skill。但**别一次性贴超过 5 条**，否则容易触发微信限流。

## 6. 排查指南（给搭建者）

skill 跑不通时，直接用脚手架命令本地排查：

```powershell
# 跑一次完整流程，并保留中间产物
python ~/.claude/skills/wechat2lark/scripts/convert.py `
  --url "https://mp.weixin.qq.com/s/xxx" `
  --config ~/.claude/skills/wechat2lark/config.json `
  --keep-temp
```

`--keep-temp` 会保留临时目录（含下载的图片和生成的 docx），打印路径在 stderr。可以拿这个 docx 直接用 Word 打开，看到底是「构 docx 阶段」就排版坏了，还是「飞书导入」阶段被改样式。

### 子命令分别验证

| 阶段 | 命令 |
|---|---|
| 抓取 + 解析 | 看脚本输出的 `image_count` 和 `content_length` 是否合理 |
| 本地 docx | `--keep-temp` 后用 Word 打开临时目录里的 .docx |
| 导入飞书 | `lark-cli drive +import --file <local.docx> --type docx --name test` |
| 挂 wiki | `lark-cli wiki +move --obj-type docx --obj-token <token> --target-space-id <id> --target-parent-token <node>` |

## 7. 维护者备忘

### 给团队加成员

```powershell
lark-cli wiki +member-add `
  --space-id 7644455953701718997 `
  --member-id <对方 open_id> `
  --member-role member
```

拿 open_id：`lark-cli contact +search-user --query "对方姓名"`

### 改父节点（比如想分年份归档）

更新所有人的 `config.json` 里的 `parent_node_token` 即可；也可以在 skill README 里改默认推荐值。

### 升级 skill

```bash
cd ~/.claude/skills/wechat2lark
git pull
```

或者重新跑 `npx skills add Jim4546/wechat2lark` 也行。
